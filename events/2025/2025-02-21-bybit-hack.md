# Bybit 冷钱包被盗事件（2025-02）

> **事件类型**: exchange_hack（交易所攻击）
> **金额**: 约 15 亿美元（401,346 ETH + 衍生品仓位）
> **攻击者**: 朝鲜 Lazarus Group（联合国已认定）
> **状态**: 部分资金被追回，大部分已分散洗钱

---

## 📋 事件时间线

- **2025-02-21 UTC**: Bybit CEO Ben Zhou 直播确认 ETH 冷钱包异常转账
- **2025-02-22**: 链上侦探 ZachXBT 等人开始追踪资金流向
- **2025-02-23**: 安全公司初步归因 Lazarus Group
- **2025-02-24**: Bybit 通过 BTC 现货 ETF、桥接资金等渠道补足客户资金缺口
- **2025-03 至今**: 持续追踪中，部分 ETH 通过 THORChain 兑换为 BTC

## 🔍 攻击原理

**核心漏洞**：Safe{Wallet} 多签钱包的**前端 UI 被入侵**

攻击流程：
1. Lazarus 攻击 Safe{Wallet} 的开发服务器
2. 注入恶意 JavaScript 到 Bybit 的多签操作界面
3. 当 Bybit 运营人员发起"真实"转账时，前端显示的收款地址是 Bybit 自己的
4. 实际链上交易的目标地址被替换为攻击者地址
5. **硬件钱包屏幕上显示的地址与屏幕被攻击者篡改**（理论上不应该，但 Safe{Wallet} 的实现有问题）
6. 多签人盲签，攻击完成

**关键点**：这不是私钥泄露，也不是智能合约漏洞 —— 是**前端/UI 层供应链攻击**。

## 💰 资金流向

- 401,346 ETH 一部分通过 DEX 分散
- 一部分通过 THORChain 跨链换成 BTC
- 一部分存入混币器（Wasabi 等）

## 🎓 学习要点

### 对交易所 / 资金管理者

1. **冷钱包不等于绝对安全**：硬件钱包 + 多签 + 离线存储也会被前端攻击绕过
2. **多签过程必须严格核对**：每位签名者都必须在硬件钱包屏幕**逐字节核对**最终交易
3. **避免使用托管前端的多签 UI**：自部署 Safe 前端或使用命令行多签工具
4. **交易模拟 + 白名单地址**：每笔转账先模拟，并限制只允许转给白名单地址
5. **分散风险**：单一冷钱包的金额不应超过总资产的一定比例

### 对普通用户

1. **大额资产不要放在一家交易所**：哪怕是头部
2. **CEX 存款 = 信任托管方**：Custodial risk 永远存在
3. **学会用硬件钱包**：Trezor / Ledger + 自己保管助记词
4. **关注交易所的 Proof of Reserves**：但 POR 不是 100% 证明

### 对开发者 / 协议设计者

1. **前端是攻击面**：任何 Web 应用都可能成为供应链攻击目标
2. **链下签名 ≠ 安全**：前端篡改可以"合法"地请求用户签错误内容
3. **EIP-712 结构化签名**：让用户在签名前看到清晰的意图，但前提是 UI 没被篡改
4. **硬件钱包的"安全屏幕"是关键**：必须使用 Ledger/Trezor 自己的屏幕显示地址
