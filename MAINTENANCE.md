# 仓库维护指南

> 维护这个仓库的基本原则和节奏

---

## 🎯 仓库定位

**核心目标**：成为 GitHub 上学习区块链最全的中文资源仓库
**关键差异化**：质量评分系统 + 个人学习笔记 + 热点事件深度分析

---

## 🚦 资源评分系统

每条资源必须带一个评分标签（4 档）：

| 标签 | 含义 | 何时用 |
|---|---|---|
| 🌟 **必读** | 不看会错过关键认知 | 行业里程碑、白皮书、深度教程 |
| ✅ **推荐** | 值得花时间看 | 系统性课程、深度文章 |
| 💡 **一般** | 闲暇时翻翻 | 入门科普、补充资料 |
| ⏸️ **过时** | 之前好，现在被超越 | 标记但保留供历史参考 |

### ⏱️ 学习时间标签

| 标记 | 含义 |
|---|---|
| ⏱️ <10min | 短文 / 快讯 |
| ⏱️ 30min | 中等文章 / 章节 |
| ⏱️ 1-2h | 视频课章节 / 单教程 |
| ⏱️ 1d+ | 系统课程 / 整本书 |

---

## 📊 资源数量限制

每个主章节（`docs/01` 到 `docs/12`）：

- **主表最多 5-10 个顶级资源**（带 🌟 / ✅）
- **完整列表**放在 `docs/XX-full-list.md` 附录中
- **不删除旧资源**（除非是过时的链接），只是移入附录

---

## 📅 维护节奏

### 每月 1 号：定期 review

- [ ] 检查所有资源链接是否仍然有效
- [ ] 检查是否有更新的版本 / 替代品
- [ ] 标记过时的资源为 ⏸️
- [ ] 更新 README 顶部"本月推荐"

### 每季度一次：结构性 review

- [ ] 主章节是否仍然合理？
- [ ] 是否需要新增 / 合并 / 删除章节？
- [ ] 事件分析章节是否覆盖了最新重大事件？

### 每年一次：整体重新审视

- [ ] 仓库目标是否仍然成立？
- [ ] 是否需要调整定位？

---

## 📖 学习笔记（notes/）规则

`notes/` 目录收录个人 / 贡献者的学习笔记。

### 谁能写

- 任何人（贡献者）
- 不要求专业背景，鼓励新人写"新手视角"

### 文件命名

```
YYYY-MM-看完-<资源名>-后.md
```

### 必须包含

1. 资源链接
2. 3 个核心认知
3. 疑问 / 不认同的地方
4. 跟其他资源的关联

### 详细模板

见 [`notes/_template.md`](_template.md)

---

## 🔥 事件分析（events/）规则

新事件分析文档应放在 `events/YYYY/` 下，按发生年份归档。

### 事件类型

| 类型 | emoji |
|---|---|
| exchange_hack（交易所攻击） | 🔓 |
| exchange_collapse（交易所倒闭） | 💥 |
| defi_hack（DeFi 协议攻击） | 🐛 |
| market_event（市场行情事件） | 📉 |
| other | ⚠️ |

### 模板

- [`events/_template/market-event.md`](_template/market-event.md) — 行情类
- 历史事件的 free form（参考 Bybit / FTX 等）

---

## 🚧 贡献指南

欢迎任何 PR，但请遵守：

1. **资源真实可用**：链接能打开
2. **中文资源优先**（这是中文资源仓库的定位）
3. **附评分 + 时间标签**
4. **简短说明为什么推荐**（一句话即可）
5. **不删除别人贡献的资源**（除非明显错误或链接失效）
